# Package Instructions

These instructions apply under `packages/`.

- Keep package boundaries strict and explicit.
- Use exact dependency versions and approved workspace protocols.
- Do not import another package's private files.
- Do not duplicate raw design tokens outside the token package.
- Add or update unit tests for behavior changes.
- Document public package APIs through exports and relevant docs.
- Keep packages private unless an approved release decision changes that status.
- Do not add runtime dependencies without architecture and security review.
