# Documentation Instructions

These instructions apply under `docs/`.

- Follow repository source precedence and locked requirements.
- Preserve stable identifiers in ADRs, registers, risks, dependencies, blockers, tests, and decisions.
- Do not fabricate clinical, legal, regulatory, privacy, security, payment, commercial, accessibility, or operational approvals.
- Do not delete governance history to make a status appear cleaner.
- Update the relevant register and change log when governance meaning changes.
- Keep Markdown links valid.
- Mark unresolved conditions explicitly instead of implying completion.
- Keep Phase 2 and pilot readiness separate from Phase 1 foundation status.
