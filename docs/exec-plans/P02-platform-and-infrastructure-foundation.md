## Required platform boundaries

- Backend architecture remains modular-monolith first under ADR-0005.
- REST is the Phase 2 API style.
- OpenAPI output must be generated by the API and used to generate a typed frontend client.
- Controllers must not access the database directly.
- External providers must be behind adapters.
- Vendor-specific SDK types must not enter domain models, API contracts, browser state, or test fixtures.
- Request and correlation IDs must flow through API, worker, logs, traces, metrics, queue jobs, and outbox records.
- Standard errors, idempotency-key middleware, database transaction helpers, transactional outbox, domain event dispatch, retry/DLQ behavior, and health/readiness endpoints are required Phase 2 conventions.
