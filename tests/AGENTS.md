# Test Instructions

These instructions apply under `tests/`.

- Use synthetic data only.
- Include negative authorization and privacy-boundary tests where relevant.
- Inspect browser console, failed requests, network payloads, storage, and cache for browser-facing work.
- Include accessibility and keyboard/focus evidence where UI is affected.
- Check reduced-motion behavior where Motion is affected.
- Add provider-disclosure privacy assertions for pharmacy or laboratory discovery work.
- Keep artifacts in ignored directories and inspect them before sharing.
- Do not weaken tests to make a gate pass.
